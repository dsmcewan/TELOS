# Historical records (superseded)

These are dated design specs, plans, and progress logs from earlier TELOS phases, kept for provenance.
They are **superseded records, not current design** — they describe the state of the system at the
time they were written and are not maintained against the shipping code. For current status see
`docs/STATUS.md`; for the protocols the gate enforces see `contracts/`.

- `specs/` — dated design specs (telos upgrade, agentic teams, provider-native outputs, situational
  awareness, ai-forge phases A–C.2).
- `plans/` — the corresponding dated implementation plans.
- `superpowers/` — earlier `plans/` + `specs/` working notes.
- `ENGINE-APPLY.md`, `sdd-progress.md` — the apply-instructions and progress/findings ledger from the
  original telos-upgrade staging (references to `me/codex/` / `ENGINE.patch` in these are historical).
