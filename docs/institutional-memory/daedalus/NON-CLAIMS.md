# Daedalus — non-claims

- **Daedalus does not authorize.** A converged workshop grants *submission* to TELOS
  authorization, nothing more. The lineage preserves two refusals of converged output
  (authz-004 on v11, authz-007 on v14) precisely so a future model sees the separation
  is real, not ceremonial.
- **Daedalus does not implement.** Carrying an authorized plan through implementation,
  verification, and documentation is *Argo's* registered role.
- **Daedalus does not validate plan truth.** The state machine checks process
  structure (objections accounted, provenance real/distinct, descent exact, coverage
  bijective) — not that the plan's content is correct.
- **No autonomous Daedalus service exists.** Workshops are explicitly invoked runs
  under `docs/runs/clotho-daedalus*/`; "Daedalus" names a registered role + protocol,
  not a daemon.
- **Daedalus cannot amend an Eye ruling.** A ruling entering a delta is a fixed,
  non-challengeable input the workshop integrates faithfully (see delta-14's
  "FIXED, NON-CHALLENGEABLE" directives).
