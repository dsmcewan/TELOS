# TELOS (role) — non-claims

- **Authorization does not execute code.** It certifies merge-readiness; carrying
  the plan through implementation is Argo.
- **A model approval is not human authority.** A unanimous signed council is still a
  model outcome; The Eye's authority is non-delegable and never inferred from
  consensus or silence.
- **The gate cannot prove an id is genuine from disk.** It can prove a `response_id`
  is present, well-formed, unique across seats, and HMAC-bound — not that the
  provider actually issued it. That binding happens at generation time.
- **A pass is not a correctness proof.** `gate_status: pass` certifies exactly what
  the checks check, nothing more.
- **The role is not the repo.** Products in this repository (ai-forge, saas-forge,
  forge) are governed beside the spine (AM-40); do not infer role ownership from
  the repository name.
