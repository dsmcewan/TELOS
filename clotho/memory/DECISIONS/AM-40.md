# Decision AM-40 — PACKAGE_ROOTS is the TELOS spine, with an explicit tested exclusion

- **what** — `PACKAGE_ROOTS` is EXACTLY the five TELOS-spine packages
  `["breakout", "build-gate", "clotho", "connectors/ai-peer-mcp", "merkle-dag"]`;
  the three sibling products `["ai-forge", "forge", "saas-forge"]` are an explicit
  committed `PACKAGE_ROOTS_EXCLUDE`. A unit mechanically discovers every **tracked**
  `package.json` directory and requires `discovered == PACKAGE_ROOTS ∪
  PACKAGE_ROOTS_EXCLUDE` and `PACKAGE_ROOTS ∩ PACKAGE_ROOTS_EXCLUDE == ∅`.
- **why** — Filesystem proximity does not establish architectural membership. Clotho
  Phase 1 weaves TELOS itself, not every product physically present in the monorepo.
  Nothing is silently omitted; a newly added package fails the suite until consciously
  classified; membership cannot drift by a directory appearing.
- **scope** — the closed `PACKAGE_ROOTS` inventory of `clotho/inventory.mjs`.
- **authority** — The Eye's ruling **AM-40**, matured by Daedalus into plan **v13**,
  authorized by **authz-006**; reaffirmed in the active plan **v15**
  (`sha256:05a48700…`, authz-008). Doc: `docs/clotho-phase-1-plan-amendments-12.md`.
- **evidence** — contract `clotho/memory/CONTRACTS/package-roots.json`; oracle
  `clotho/scripts/test-inventory.mjs` (completeness/disjointness) + `verify-contracts.mjs`
  (contract == `clotho/inventory.mjs` PACKAGE_ROOTS/EXCLUDE).
- **non_claim** — not a claim that these are the only possible packages; it is the
  closed, human-authorized set. Enrolling a product requires a deliberate amendment.
- **change_rule** — expanding/contracting the set requires a plan amendment (Daedalus),
  TELOS re-authorization, implementation review, and updated completeness fixtures.

**Origin (institutional memory):** this ruling exists because a model reading eight
`package.json` directories "helpfully" included all eight. The exclusion is now
mechanically enforced *and* gated by `clotho/memory/comprehension-queries.json`.
