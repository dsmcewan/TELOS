# Decision AM-41 — the D33 scanner is correct over an enforced closed source profile

- **what** — the shared D33 classifier/resolver (`clotho/weavers/util.mjs`, used by both
  the closure derivation and the advisory scanner) is required correct ONLY over a
  **closed, mechanically enforced source profile**, and must reject every out-of-profile
  construct **deterministically** with the stable diagnostic
  `unsupported-module-lexical-profile`. The exact out-of-profile set is **b1–b6**
  (bare CR/U+2028/U+2029; non-leading hashbang; HTML/`-->`; string
  line-continuation/octal escape; string-literal specifier name; unterminated/truncated
  form). **One** optional leading shebang (`#!` at byte 0, first line, LF/CRLF-terminated)
  is admitted and stripped before lexing; a `#!` inside a string or comment is not a
  hashbang.
- **why** — a hand-written lexer met a maximally-thorough required seat and defects
  recurred for rounds. The Eye ruled the fix is **not** a full ECMAScript parser and
  **not** an override of the seat, but *"total correctness over an explicit subset that
  refuses everything else"* — provably correct in-profile, fail-closed out-of-profile.
- **scope** — the shared D33 scanner over committed, trusted Clotho ESM source.
- **authority** — The Eye's ruling **AM-41**, matured by Daedalus into **v14** (+ the
  shebang carve-out in **v15**), authorized by **authz-008** (`sha256:05a48700…`). Doc:
  `docs/clotho-phase-1-plan-amendments-13.md`.
- **evidence** — contract `clotho/memory/CONTRACTS/source-profile.json`; oracle
  `clotho/scripts/test-util.mjs` + `test-closure.mjs` (supported-profile correctness +
  each b1–b6 fails closed); safety net = the closure-equality assertion.
- **non_claim** — not a complete ECMAScript parser; not a sandbox; does not prove loader
  containment (`clotho/memory/NON-CLAIMS.md`).
- **change_rule** — any change to the profile or the out-of-profile set requires a plan
  amendment, TELOS re-authorization, implementation review, and updated adversarial
  fixtures.

**Institutional-memory note.** `authz-007` correctly returned **NOT_AUTHORIZED** for v14
because AM-41's blanket hashbang rejection contradicted Task 5's requirement that
`clotho/weave.mjs` (a closure entry point) carry a Node shebang. The Eye resolved it with
the **leading-shebang carve-out**, integrated in v15, authorized by authz-008. That
NOT_AUTHORIZED record is preserved as evidence that authorization catches plan
contradictions before implementation.
